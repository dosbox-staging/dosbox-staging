# PR 5 — single-source shader library (execution spec)

Part of the native Vulkan render backend series. Rationale in
`native-gpu-backends-plan.md`: decision 4 (single-source shaders,
offline GL generation), the “Offline GL 330 generation” section
(spike-verified pipeline — follow it exactly), Appendix D Spike 1
(the verbatim CLI round-trip this script automates), and flesh-out
decision 3 (directory layout). **If reality disagrees with this
spec, stop and update the spec first.**

## Goal

Collapse the shader sources to ONE hand-maintained tree: Vulkan GLSL
4.50 under `resources/shaders/`, with the OpenGL 3.30 variants
generated offline into `resources/shaders/gl330/`, checked in, and
guarded by a CI `--check` so they can never go stale. The OpenGL
executor switches to flattened uniform upload (shared UniformPacker
bytes), deleting the by-name uniform paths. PR 3's temporary
`vulkan450/` tree disappears.

## Prerequisites

- PR 3 merged (8 shaders already 450-ported; compiler + packer live;
  per-backend path resolution mechanism exists).

## Success criteria (PR level)

1. `resources/shaders/` contains only 450 sources (21 files);
   `resources/shaders/gl330/` is a complete generated mirror; the
   `vulkan450/` tree is gone.
2. Regenerating on a second machine produces zero diffs
   (deterministic output, pinned tools).
3. GL golden captures: legacy uniform path vs flattened path are
   **bit-identical** per shader (same machine, same GPU).
4. CI fails if a 450 source changes without its regenerated 330
   mirror (`--check` hook), and the generation script is
   pylint-clean (`./scripts/linting/verify-python.sh`).
5. `crt-auto-*` works on all three backends from the new trees.
6. Every commit standalone-green.

## Commits

### Commit 1 — `Add the offline GL 3.30 shader generation script`

**New files:** `scripts/tools/generate-gl-shaders.py`; CI hook (see
step 5).

**Steps:**

1. Implement the Appendix D pipeline per shader, exactly:
   1. `glslang -V -DVERTEX=1 -S vert -o v.spv <src>` and
      `-DFRAGMENT=1 -S frag -o f.spv <src>`;
   2. `spirv-cross --version 330 --no-es --flatten-ubo <stage>.spv`
      (add `--flip-vert-y` for the vertex stage ONLY if commit 2's
      golden tests prove GL needs it — start without);
   3. post-process: strip `layout(binding = N)` from sampler
      declarations (420pack-only — regex on
      `^layout\(binding = \d+\) (uniform sampler2D)` → `\1`);
      optionally strip the harmless 420pack `#ifdef` header;
   4. stitch the two stages back into the single-file
      `#if defined(VERTEX)` / `#elif defined(FRAGMENT)` format with
      the original `#pragma` comment block re-attached byte-
      identically (extract it from the 450 source before step i);
   5. write to `resources/shaders/gl330/<same subpath>`.
2. **Tool pinning (determinism guardrail):** the script locates
   `glslang` and `spirv-cross` binaries from, in order: explicit
   `--glslang`/`--spirv-cross` flags, the vcpkg host-tools dir under
   the build tree, then `$PATH`. It embeds a
   `// Generated by generate-gl-shaders.py — glslang X.Y, SPIRV-Cross Z. DO NOT EDIT.`
   header line in every output and **fails `--check` with a
   version-mismatch message** (not a content diff) when the local
   tools differ from the versions recorded in the checked-in files —
   distinguishing “stale files” from “different toolchain”.
   Add `glslang` (already a dep) and `spirv-cross` as **host** vcpkg
   dependencies so CI and devs share pinned tool builds — spirv-cross
   is dev-time only; it must NOT appear in any link line
   (decision 5).
3. `--check` mode: regenerate to a temp dir, diff against the
   checked-in tree, non-zero exit + unified diff on any change.
4. Determinism self-test: run generation twice, diff — identical.
5. CI hook: **into the Linux build workflow**, not linting.yml — the
   lint job has no vcpkg toolchain, and pinned tools are the whole
   point. Add a step to one linux.yml matrix entry after the build:
   `python3 scripts/tools/generate-gl-shaders.py --check
   --tools-from build/<preset>`. (This diverges from the plan's
   one-line “lint workflow gains the hook” for a verified reason —
   the plan text is updated alongside this PR.)
6. `./scripts/linting/verify-python.sh` must pass — write the script
   pylint-clean from the start.

**Automated verification:** script run over the 8 existing 450
sources produces 330 output that compiles on the GL backend (boot
probe with each, `output = opengl`, temporary manual pointing of the
resolver — or simply hold this until commit 3 wires the tree and
verify the 8 there); determinism self-test; `--check` green then
deliberately edit one 450 file → `--check` must fail → revert.

**Attribution flips:** none.

### Commit 2 — `Switch the OpenGL executor to flattened uniform blocks`

**Files:** `opengl/private/opengl_pipeline.*`,
`opengl/private/opengl_shader.*`.

**Steps:**

1. The GL pipeline uploads each pass's uniforms with ONE
   `glUniform4fv(loc, k, packed_bytes)` into the flattened
   `Uniforms[k]` vec4 array (the flatten name derives from the block
   *type* and is stable — Spike 1 finding 3), sourcing bytes from
   the same `Shaders::UniformPacker` the Vulkan executor uses.
   `k = BlockSizeBytes() / 16` rounded up.
2. Delete the by-name `OpenGlShader::SetUniform*` paths and their
   call sites. Sampler binding by preserved name at link time stays.
3. This commit must work with BOTH source trees: the legacy 330
   hand-written files (still current for the 13 unported shaders)
   AND generated-330 files. Hand-written 330 shaders still declare
   loose uniforms — so this commit ALSO regenerates the 8 ported
   shaders' 330 variants into `gl330/` (using commit 1's script) and
   points the GL resolver at `gl330/` for exactly those 8 (the
   resolver mechanism from PR 3 commit 6). The 13 unported shaders
   keep the legacy path until commit 3. Keep the legacy uniform
   upload code alive behind the source-tree distinction until
   commit 3 removes the last user, then delete it there.
4. `--flip-vert-y` decision: capture a golden with the generated
   `sharp` — if the image is vertically flipped vs the legacy
   capture, add `--flip-vert-y` to the script's vertex-stage
   invocation (commit 1's flag exists for this), regenerate,
   recapture. Record the outcome in the commit message and the
   learning doc.

**Automated verification & the bit-identical gate:** for each of
the 8 ported shaders: capture (Ctrl+F5) a fixed test screen on the
legacy path (pre-commit build), then on the flattened path
(post-commit build), `cmp` — **bit-identical required** (same
machine, same GPU; this is the plan's explicit standard for
GL-legacy vs GL-generated). Script the loop:
build A → 8 captures → build B → 8 captures → `cmp` ×8. Full ctest
both builds.

**Attribution flips:** none.

### Commit 3 — `Convert the bundled shader library to the 450 format`

**Files:** the remaining 13 shader sources (3 crt/, 3 interpolation/
minus sharp, 7 scaler/), the tree inversion, resolver update,
deletion of the PR 3 `vulkan450/` tree and of the legacy GL uniform
upload path.

**Steps:**

1. Port the 13 remaining shaders with the same per-shader loop as
   PR 3 commit 6 (recipe + reserved-word audit + compile test +
   eyeball). The CRT and xbr shaders are the big ones — take them
   one at a time; the compiler unit test's data-driven enumeration
   picks each up automatically.
2. Invert the trees: 450 sources move to `resources/shaders/`
   (replacing their 330 originals — `git mv` the 450 file over,
   delete the 330); run the generation script → full
   `resources/shaders/gl330/` mirror, checked in; delete
   `resources/shaders/vulkan450/`; update `ShaderManager`'s
   per-backend resolution (vulkan → main tree, opengl → `gl330/`);
   delete the now-unused legacy GL uniform path from commit 2 step 3.
3. User shader directories are untouched: user files are read from
   the same locations as today and assumed to be in the active
   backend's format (plan decision 15).
4. The resource-copy mechanism (`cmake/add_copy_assets.cmake`) globs
   recursively — no build changes needed for the new tree; verify
   `gl330/` lands in the build's resources dir.

**Automated verification:** `--check` green; compiler test covers
all 21 files; goldens: lavapipe records for the newly-ported CRT +
one scaler shader; GL captures per shader family (crt, scaler,
interpolation) vs pre-PR legacy captures — bit-identical;
`crt-auto` boot probes on all three backends; full ctest.

**Manual verification:** `crt-auto-*` variants on real 13h/text/VESA
content on all three backends; spot-check xbr shaders (they push the
compiler hardest).

**Attribution flips:** none.
