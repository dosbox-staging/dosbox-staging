Real hardware OPL chip passthrough
Allows you to use a real hardware OPL chip. To activate, set oplmode=hardware and hardwarebase= to the address value of your real soundblaster in dosbox.conf. On Windows this feature reqires the porttalk driver, see the top of this page.

Serial FIFO support and logging capability enabled
It provides logging for the serial port. The log is stored in the subdirectory capture. It's useful for finding bugs in the emulation and for reverse-engineering old serial protocols. To activate, add some of the following terms to the serialx config options:
  • dbgtr - adds traffic-related messages to the log
  • dbgirq - interrupt messages
  • dbgreg - register accesses from the DOS application
  • dbgmd - changes on the modem control lines
  • dbgaux - serial device specific messages
Example:
serial1=directserial reaport:com1 dbgirq dbgreg dbgtr dbgmd
