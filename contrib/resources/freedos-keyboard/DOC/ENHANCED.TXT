Support for official national/regional languages
================================================


The keyboards described in this document had their functionality enhanced for
at least one of the following reasons:

a) Official language(s) they are meant to assist is(are) not fully  supported
   or not supported at all.

b) There were user requests for supporting (a) particular language(s).
   
c) There is(are) (an-)other major-industry codepage(s) suitable for them.

Nevertheless, it is important to  explain  that  all  enhancements  are  only
accessed when the user is NOT using the default codepage. This  is  to  avoid
sudden, unexpected discomfort when using them with their default codepage.

There is an exception to case  (a) : Russia. There  are  many  more  official
languages than what could be assisted into a single KEY file. A "RX.KEY" file
("Russian international") was devised to assist that case and the details are
in the "RUSSIA.TXT" file available on the documentation.

Furthermore, it is expected (even though the mention to this case goes beyond
the scope of this document), due to  cultural  ties  with  former  colonizing
countries, that former colonies use the same keyboards (or at least sell them
as an alternative to a keyboard devised on their own or to the US  keyboard).
Fall in this case almost all non-european  countries. They  have  nonetheless
other official languages and it would be impossible, on most cases, to encode
all of them into a single keyboard layout. For example, encoding all official
languages of the former french colonies into the french keyboard would not be
viable. To solve such cases, "international" versions of the original french,
UK, US, spanish, portuguese and belgian layouts were devised. Please refer to
their particular documentation ("xx-INTL.TXT") for details.
