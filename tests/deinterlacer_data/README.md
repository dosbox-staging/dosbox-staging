# Deinterlacer golden-image test fixtures

Reference images for the golden tests in `tests/deinterlacer_tests.cpp`
(the `DeinterlacerGoldenTest` suite). Each test loads an input frame
from `inputs/`, runs it through the FMV deinterlacer, and compares the
result against the matching file in `expected/`.

## Input frames

The current inputs are SYNTHETIC PLACEHOLDERS, generated by
`tools/make_synthetic_inputs.py` (Python standard library only). They
mimic the two fake-interlacing schemes the deinterlacer handles:

- `line-640x480.png` — alternating content/black scanlines
  (640x480 FMV titles, line deinterlacing)
- `dot-320x200.png` — the KGB/Dune dot pattern (mode 13h, dot
  deinterlacing)
- `dot-320x400.png` — the same frame with baked-in double scanning
  (every row doubled), the real shape of a mode 13h capture
- `dot-640x400.png` — width- and height-doubled variant

They are meant to be replaced with real FMV captures. When swapping
them in, keep the constraints: widths must be a multiple of 64 (the
deinterlacer processes rows in 64-pixel bit-buffer chunks), the line
frame's video mode must be at least 400 lines tall with 256+ colours,
and the dot frames must be BIOS mode 13h shaped (the doubled variants'
pixel dimensions must be exact integer multiples of 320x200). After
swapping, regenerate the references (below) and inspect them.

## Regenerating references

After an intentional behaviour change or an input-frame swap:

1. Delete the affected `.png` files in `expected/`.
2. Run the tests:
   `build/<preset>/tests/dosbox_tests --gtest_filter='DeinterlacerGoldenTest.*'`
3. The test harness writes the new candidates to `expected/` and
   fails — that's expected.
4. **Inspect the candidates visually.** They're committed-as-output
   and CI compares against them; a wrong reference passes forever.
5. `git add` the new references and commit.

On a mismatch the produced image is written next to the reference as
`<name>.png-ACTUAL.png` (gitignored) for side-by-side inspection.

## Tolerance

The deinterlacer is deterministic integer math, so references are
compared exactly (zero tolerance).
